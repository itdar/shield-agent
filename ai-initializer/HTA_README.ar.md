# نظام إدارة وكلاء الذكاء الاصطناعي — دليل للبشر

## كيف تبدأ؟

> **تحذير: استهلاك عالي للرموز (Tokens)** — يستخدم الإعداد الأولي نموذجاً من المستوى الأعلى لتحليل المشروع بالكامل وإنشاء عدة ملفات (AGENTS.md، .ai-agents/context/، .ai-agents/skills/، .ai-agents/roles/). توقع استهلاكاً كبيراً للرموز (عشرات الآلاف أو أكثر حسب حجم المشروع). هذه تكلفة لمرة واحدة — الجلسات اللاحقة تحمّل السياق المُعدّ مسبقاً وتبدأ فوراً.

```bash
# 1. اجعل الذكاء الاصطناعي يقرأ HOW_TO_AGENTS.md وسيقوم بالإعداد تلقائياً

# الخيار أ: الإنجليزية (موصى به — تكلفة رموز أقل، أفضل أداء للذكاء الاصطناعي)
claude --dangerously-skip-permissions --model claude-opus-4-6 \
  "Read HOW_TO_AGENTS.md and generate AGENTS.md tailored to this project"

# الخيار ب: لغتك (موصى به إذا كان البشر يحررون AGENTS.md مباشرة)
claude --dangerously-skip-permissions --model claude-opus-4-6 \
  "Read HOW_TO_AGENTS.md and generate AGENTS.md tailored to this project (in Arabic)"

# موصى به: --dangerously-skip-permissions للإعداد التلقائي بدون انقطاع
# موصى به: --model claude-opus-4-6 (أو أحدث) للحصول على أفضل النتائج

# 2. ابدأ العمل مع الوكلاء المُنشأين
./ai-agency.sh

# 3. من الجلسات التالية فصاعداً، يمكنك البدء بالعمل فوراً وتلقائياً!
```

> هذا المستند مُعدّ **ليقرأه ويفهمه البشر**.
> يشرح لماذا يوجد دليل التعليمات الذي ينفذه الذكاء الاصطناعي (HOW_TO_AGENTS.md)،
> وكيف يعمل من حيث المبدأ، وما هو دوره في سير عمل التطوير الخاص بك.

## المبدأ الأساسي: هيكل من 3 طبقات

```
                    مشروعك
                         │
            ┌────────────┼────────────┐
            ▼            ▼            ▼

     ┌──────────┐  ┌──────────┐  ┌──────────┐
     │ AGENTS.md│  │.ai-agents/context│  │.ai-agents/skills│
     │          │  │          │  │          │
     │ الهوية   │  │ المعرفة  │  │ السلوك   │
     │ "من      │  │ "واجهات  │  │ "عند     │
     │  أنا؟"   │  │  هذه     │  │  التطوير │
     │          │  │  الخدمة  │  │  افعل    │
     │ + قواعد  │  │  هكذا"   │  │  هكذا"   │
     │ + صلاحيات│  │          │  │          │
     │ + مسارات │  │ + النطاق │  │ + النشر  │
     │          │  │ + النموذج│  │ + المراجعة│
     └──────────┘  └──────────┘  └──────────┘
       نقطة الدخول   مخزن الذاكرة   معايير سير العمل
```

### 1. AGENTS.md — "من أنا؟"

هو **ملف الهوية** للوكيل الموضوع في كل مجلد.

```
المشروع/
├── AGENTS.md                  ← مدير المشروع: القائد الذي ينسق الكل
├── apps/
│   └── doppel-api/
│       └── AGENTS.md          ← خبير API: مسؤول عن هذه الخدمة فقط
├── infra/
│   ├── AGENTS.md              ← SRE: إدارة البنية التحتية بالكامل
│   └── monitoring/
│       └── AGENTS.md          ← خبير المراقبة
└── configs/
    └── AGENTS.md              ← مدير الإعدادات
```

إنه يشبه **الهيكل التنظيمي للفريق**:
- مدير المشروع يرى الصورة الكاملة ويوزع المهام
- كل عضو في الفريق يفهم مجاله بعمق فقط
- لا يقوم بعمل فريق آخر مباشرة بل يطلب منه

### 2. `.ai-agents/context/` — "ماذا أعرف؟"

هو مجلد يحتوي على **المعرفة الأساسية المُرتبة مسبقاً** حتى لا يحتاج الذكاء الاصطناعي لقراءة الكود في كل مرة.

```
.ai-agents/context/
├── domain-overview.md     ← "هذه الخدمة مسؤولة عن إدارة الطلبات..."
├── data-model.md          ← "توجد كيانات Order، Payment، Delivery..."
├── api-spec.json          ← "POST /orders, GET /orders/{id}, ..."
└── event-spec.json        ← "تُصدر حدث order-created..."
```

**تشبيه:** إنه مثل وثيقة الإعداد الأولي (Onboarding) التي تُعطى للموظف الجديد.
"ماذا يفعل فريقنا، كيف هيكل قاعدة البيانات، ما هي الـ APIs المتاحة" — بمجرد ترتيبها مرة واحدة
لن تحتاج لشرحها في كل مرة.

### 3. `.ai-agents/skills/` — "كيف أعمل؟"

هو **دليل سير العمل المعياري** للمهام المتكررة.

```
.ai-agents/skills/
├── develop/SKILL.md       ← "تطوير ميزة: تحليل → تصميم → تنفيذ → اختبار → PR"
├── deploy/SKILL.md        ← "النشر: وسم → طلب → تحقق"
└── review/SKILL.md        ← "المراجعة: قائمة تحقق الأمان والأداء والاختبارات"
```

**تشبيه:** إنه دليل عمل الفريق.
"عند رفع PR تحقق من هذه القائمة" — نجعل الذكاء الاصطناعي يتبع ذلك أيضاً.

## كيف تُدار القواعد العامة؟

يُستخدم **نمط الوراثة**. تُكتب في مكان واحد وتُطبق تلقائياً على ما تحته.

```
AGENTS.md الجذر ──────────────────────────────────────────
│ Global Conventions:
│  - الالتزام: Conventional Commits (feat:, fix:, chore:)
│  - PR: القالب إلزامي، مراجعة شخص واحد على الأقل
│  - الفرع: feature/{ticket}-{desc}
│  - الكود: TypeScript strict, single quotes
│
│     وراثة تلقائية                وراثة تلقائية
│     ┌──────────────────┐       ┌──────────────────┐
│     ▼                  │       ▼                  │
│  apps/api/AGENTS.md    │    infra/AGENTS.md       │
│  (تُذكر القواعد         │    (تُذكر القواعد         │
│   الإضافية فقط)         │     الإضافية فقط)         │
│  "هذه الخدمة            │    "عند تغيير Helm values │
│   تستخدم Python"        │     اسأل أولاً"           │
│     (بدلاً من TypeScript)│                          │
└─────────────────────────┴──────────────────────────
```

**المزايا:**
- تريد تغيير قواعد الالتزام؟ → عدّل مكاناً واحداً فقط في الجذر
- أضفت خدمة جديدة؟ → تُطبق القواعد العامة تلقائياً
- خدمة معينة مختلفة؟ → تجاوزها في AGENTS.md الخاص بتلك الخدمة

## ماذا نكتب وماذا لا نكتب؟

وفقاً لبحث ETH Zurich (2026)، إذا كتبت في المستند محتوى يستطيع الذكاء الاصطناعي استنتاجه بالفعل،
فإن **معدل النجاح ينخفض والتكلفة تزداد بنسبة 20%**.

```
             ما نكتبه                          ما لا نكتبه
     ┌─────────────────────────┐     ┌─────────────────────────┐
     │                         │     │                         │
     │  "الالتزام بصيغة feat:"   │     │  "المصدر في مجلد src/"    │
     │  ما لا يستطيع الذكاء      │     │  يمكن للذكاء الاصطناعي   │
     │  الاصطناعي استنتاجه       │     │  رؤيته بأمر ls مباشرة    │
     │                         │     │                         │
     │  "ممنوع الدفع إلى main"  │     │  "React قائم على         │
     │  "مباشرة"                │     │   المكونات"              │
     │  قاعدة فريق غير موجودة   │     │  موجود في الوثائق        │
     │  في الكود               │     │  الرسمية أصلاً           │
     │                         │     │                         │
     │  "موافقة فريق QA إلزامية │     │  "هذا الملف 100 سطر"     │
     │   قبل النشر"             │     │  يمكن للذكاء الاصطناعي   │
     │  عملية لا يمكن استنتاجها  │     │  قراءته مباشرة           │
     │                         │     │                         │
     └─────────────────────────┘     └─────────────────────────┘
          تُدوّن في AGENTS.md               ممنوع تدوينها!
```

**لكن هناك استثناء:** "ما يمكن استنتاجه لكن تكلفة تكراره عالية جداً"

```
  مثال: قائمة جميع الـ APIs (يتطلب قراءة 20 ملفاً لمعرفتها)
  مثال: مخطط علاقات نموذج البيانات (موزع على 10 ملفات)
  مثال: علاقات الاستدعاء بين الخدمات (يتطلب التحقق من الكود + البنية التحتية)

  → هذه الأشياء تُرتب مسبقاً في .ai-agents/context/!
  → في AGENTS.md نكتب فقط المسار "ستجدها هنا"
```

## سكربت تشغيل الجلسات

بمجرد إعداد جميع الوكلاء، يمكنك اختيار الوكيل المطلوب وبدء الجلسة مباشرة.

```bash
$ ./ai-agency.sh

=== AI Agent Sessions ===
Project: /Users/nhn/src/doppel-deploy
Found: 8 agent(s)

  1) [PM] doppel-deploy
     Path: ./AGENTS.md
     وكيل مدير المشروع. يستوعب الهيكل الكامل ويفوض المهام.

  2) doppel-api
     Path: apps/dev/doppel-api/AGENTS.md
     خبير إدارة مانيفست K8s لخدمة doppel-api.

  3) monitoring
     Path: infra/monitoring/AGENTS.md
     خبير SRE لحزمة المراقبة Prometheus + Grafana.

  ...

Select agent (number): 2

=== AI Tool ===
  1) claude
  2) codex
  3) print

Select tool: 1

→ بدأت جلسة claude في مجلد doppel-api
→ الوكيل يحمّل AGENTS.md و .ai-agents/context/ تلقائياً
→ يمكنك البدء بالعمل فوراً!
```

**التنفيذ المتوازي (tmux):**

```bash
$ ./ai-agency.sh --multi

Select agents: 1,2,3   # تشغيل PM + API + المراقبة في وقت واحد

→ تُفتح 3 جلسات tmux
→ في كل نافذة يعمل وكيل مختلف بشكل مستقل
→ Ctrl+B N للتبديل بين النوافذ
```

## قائمة المخرجات

الملفات المُنتجة من هذا النظام واستخداماتها:

| الملف | الجمهور المستهدف | الغرض |
|---|---|---|
| `HOW_TO_AGENTS.md` | الذكاء الاصطناعي | دليل التعليمات الوصفية الذي يقرأه الوكيل وينفذه |
| `HOW_TO_AGENTS_PLAN.md` | البشر/الذكاء الاصطناعي | خطة التصميم (خلفية سبب هذا الهيكل) |
| `README.md` | البشر | هذا المستند. دليل لفهم البشر |
| `ai-agency.sh` | البشر | مُشغّل اختيار الوكيل → بدء جلسة الذكاء الاصطناعي |
| `AGENTS.md` (كل مجلد) | الذكاء الاصطناعي | هوية الوكيل + القواعد لكل مجلد |
| `.ai-agents/context/*.md/json` | الذكاء الاصطناعي | معرفة النطاق المُرتبة مسبقاً |
| `.ai-agents/skills/*/SKILL.md` | الذكاء الاصطناعي | سير العمل المعياري |
| `.ai-agents/roles/*.md` | الذكاء الاصطناعي/البشر | استراتيجية تحميل السياق حسب الدور |

## التشبيه الجوهري

```
              فريق التطوير التقليدي       فريق وكلاء الذكاء الاصطناعي
              ──────────────────         ──────────────────────────
 القائد       مدير المشروع (بشر)         AGENTS.md الجذر (وكيل PM)
 الأعضاء      N مطور                    AGENTS.md في كل مجلد
 وثائق الإعداد  Confluence/Notion        .ai-agents/context/
 دليل العمل    ويكي الفريق               .ai-agents/skills/
 تعريف الأدوار  وثيقة المسميات/المسؤوليات  .ai-agents/roles/
 قواعد الفريق  وثيقة اتفاقيات الفريق      Global Conventions (وراثة)
 بدء العمل    الوصول للمكتب              بدء الجلسة → تحميل AGENTS.md
 انتهاء العمل  المغادرة (الذاكرة باقية)    انتهاء الجلسة (الذاكرة تتلاشى!)
 اليوم التالي  الذاكرة موجودة             تحميل .ai-agents/context/ (استعادة الذاكرة)
```

**الفرق الجوهري:** البشر يحتفظون بذاكرتهم حتى بعد مغادرة العمل، لكن الذكاء الاصطناعي ينسى في كل مرة.
لهذا السبب يوجد `.ai-agents/context/` — فهو يلعب دور **الذاكرة طويلة المدى** للذكاء الاصطناعي.
