# AI एजेंट प्रबंधन प्रणाली — मनुष्यों के लिए गाइड

## शुरू कैसे करें?

> **टोकन उपयोग चेतावनी** — प्रारंभिक सेटअप शीर्ष-स्तरीय मॉडल का उपयोग करके पूरे प्रोजेक्ट का विश्लेषण करता है और कई फ़ाइलें (AGENTS.md, .ai-agents/context/, .ai-agents/skills/, .ai-agents/roles/) बनाता है। प्रोजेक्ट के आकार के अनुसार दसियों हज़ार या अधिक टोकन की खपत हो सकती है। यह एक बार की लागत है — बाद के सत्र पूर्व-निर्मित कॉन्टेक्स्ट लोड करके तुरंत शुरू होते हैं।

```bash
# 1. AI को HOW_TO_AGENTS.md पढ़ाएं, वह खुद सेटअप कर लेगा

# विकल्प A: अंग्रेजी (अनुशंसित — कम टोकन लागत, सर्वोत्तम AI प्रदर्शन)
claude --dangerously-skip-permissions --model claude-opus-4-6 \
  "Read HOW_TO_AGENTS.md and generate AGENTS.md tailored to this project"

# विकल्प B: आपकी भाषा (अनुशंसित यदि मनुष्य सीधे AGENTS.md संपादित करते हैं)
claude --dangerously-skip-permissions --model claude-opus-4-6 \
  "HOW_TO_AGENTS.md को पढ़ो और इस प्रोजेक्ट के अनुसार AGENTS.md बनाओ (हिंदी में)"

# अनुशंसित: --dangerously-skip-permissions बिना रुकावट स्वायत्त सेटअप के लिए
# अनुशंसित: --model claude-opus-4-6 (या बाद का संस्करण) सर्वोत्तम परिणामों के लिए

# 2. बनाए गए एजेंट से काम शुरू करें
./ai-agency.sh

# 3. इसके बाद के सत्रों से स्वचालित रूप से तुरंत काम शुरू हो सकता है!
```

> यह दस्तावेज़ **मनुष्यों द्वारा पढ़ने और समझने के लिए** है।
> AI द्वारा निष्पादित निर्देश पुस्तिका (HOW_TO_AGENTS.md) क्यों मौजूद है, किस सिद्धांत से काम करती है,
> और आपके विकास कार्यप्रवाह में क्या भूमिका निभाती है, यह समझाता है।

### 1. AGENTS.md — "मैं कौन हूं"

प्रत्येक डायरेक्टरी में रखी गई एजेंट की **पहचान फ़ाइल** है।

```
प्रोजेक्ट/
├── AGENTS.md                  ← PM: पूरे प्रोजेक्ट का समन्वय करने वाला लीडर
├── apps/
│   └── doppel-api/
│       └── AGENTS.md          ← API विशेषज्ञ: केवल इस सेवा का प्रभारी
├── infra/
│   ├── AGENTS.md              ← SRE: संपूर्ण इंफ्रास्ट्रक्चर प्रबंधन
│   └── monitoring/
│       └── AGENTS.md          ← मॉनिटरिंग विशेषज्ञ
└── configs/
    └── AGENTS.md              ← कॉन्फ़िगरेशन प्रबंधक
```

यह बिल्कुल **टीम संगठन चार्ट** जैसा है:
- PM पूरे प्रोजेक्ट को देखता है और कार्य वितरित करता है
- प्रत्येक सदस्य केवल अपने क्षेत्र को गहराई से समझता है
- दूसरी टीम का काम सीधे नहीं करता, अनुरोध करता है

### 2. `.ai-agents/context/` — "क्या जानता है"

AI को हर बार कोड पढ़ने की ज़रूरत न हो इसलिए **मुख्य ज्ञान को पहले से व्यवस्थित** करके रखने वाला फ़ोल्डर है।

```
.ai-agents/context/
├── domain-overview.md     ← "यह सेवा ऑर्डर प्रबंधन का काम करती है..."
├── data-model.md          ← "Order, Payment, Delivery एंटिटी हैं और..."
├── api-spec.json          ← "POST /orders, GET /orders/{id}, ..."
└── event-spec.json        ← "order-created इवेंट प्रकाशित करती है और..."
```

**उपमा:** यह नए कर्मचारी को दिए जाने वाले ऑनबोर्डिंग दस्तावेज़ जैसा है।
"हमारी टीम क्या करती है, DB संरचना कैसी है, कौन-कौन से API हैं" — एक बार व्यवस्थित कर दो तो
हर बार समझाने की ज़रूरत नहीं।

### 3. `.ai-agents/skills/` — "कैसे काम करता है"

बार-बार दोहराए जाने वाले कार्यों को मानकीकृत करने वाला **कार्यप्रवाह मैनुअल** है।

```
.ai-agents/skills/
├── develop/SKILL.md       ← "फ़ीचर विकास: विश्लेषण → डिज़ाइन → कार्यान्वयन → परीक्षण → PR"
├── deploy/SKILL.md        ← "डिप्लॉयमेंट: टैग → अनुरोध → सत्यापन"
└── review/SKILL.md        ← "रिव्यू: सुरक्षा·प्रदर्शन·परीक्षण चेकलिस्ट"
```

**उपमा:** यह टीम का कार्य मैनुअल है।
"PR डालते समय इस चेकलिस्ट की जांच करो" — ऐसी बातें AI से भी करवाई जाती हैं।

## वैश्विक नियम कैसे प्रबंधित होते हैं?

**इनहेरिटेंस पैटर्न** का उपयोग होता है। एक जगह लिखो, नीचे स्वचालित रूप से लागू होता है।

```
रूट AGENTS.md ──────────────────────────────────────────
│ Global Conventions:
│  - कमिट: Conventional Commits (feat:, fix:, chore:)
│  - PR: टेम्पलेट अनिवार्य, 1 या अधिक रिव्यू
│  - ब्रांच: feature/{ticket}-{desc}
│  - कोड: TypeScript strict, single quotes
│
│     स्वचालित इनहेरिटेंस            स्वचालित इनहेरिटेंस
│     ┌──────────────────┐       ┌──────────────────┐
│     ▼                  │       ▼                  │
│  apps/api/AGENTS.md    │    infra/AGENTS.md       │
│  (केवल अतिरिक्त नियम)    │    (केवल अतिरिक्त नियम)    │
│  "यह सेवा               │    "Helm values बदलते    │
│   Python उपयोग करती है"  │     समय Ask First"      │
│     (TypeScript के बजाय) │                          │
└─────────────────────────┴──────────────────────────
```

**लाभ:**
- कमिट नियम बदलना है? → केवल रूट में एक जगह संशोधन
- नई सेवा जोड़ना है? → वैश्विक नियम स्वचालित रूप से लागू
- किसी विशेष सेवा को अलग रखना है? → उस सेवा के AGENTS.md में ओवरराइड

बल्कि **सफलता दर गिरती है और लागत 20% बढ़ जाती है**।

```
                लिखना चाहिए                       नहीं लिखना चाहिए

     │                         │     │                         │
     │  "कमिट feat: फॉर्मेट में"  │     │  "src/ फ़ोल्डर में सोर्स है" │

     │  "डिप्लॉय से पहले QA टीम │     │  "यह फ़ाइल 100 पंक्तियों  │
     │   की मंज़ूरी अनिवार्य"    │     │   की है"                 │
     │  प्रक्रिया, अनुमान संभव   │     │  AI सीधे पढ़ सकता है     │
     │  नहीं                    │     │                         │
     └─────────────────────────┘     └─────────────────────────┘
             AGENTS.md में दर्ज                  दर्ज मत करो!
```

**लेकिन एक अपवाद है:** "अनुमान तो लगाया जा सकता है लेकिन हर बार लगाना बहुत महंगा है"

```
  उदाहरण: संपूर्ण API सूची (20 फ़ाइलें पढ़नी पड़ती हैं)
  उदाहरण: डेटा मॉडल संबंध चित्र (10 फ़ाइलों में बिखरा हुआ)
  उदाहरण: सेवाओं के बीच कॉल संबंध (कोड + इंफ्रा दोनों जांचने की ज़रूरत)

  → ऐसी चीज़ें .ai-agents/context/ में पहले से व्यवस्थित करो!
  → AGENTS.md में केवल "यहां जाओ तो मिलेगा" पथ लिखो
```

## सत्र लॉन्चर स्क्रिप्ट

सभी एजेंट सेटअप हो जाने पर, आप चाहे जो एजेंट चुनकर तुरंत सत्र शुरू कर सकते हैं।

```bash
$ ./ai-agency.sh

=== AI Agent Sessions ===
Project: /Users/nhn/src/doppel-deploy
Found: 8 agent(s)

  1) [PM] doppel-deploy
     Path: ./AGENTS.md
     इस प्रोजेक्ट का PM एजेंट। पूरी संरचना समझता है और कार्य सौंपता है।

  2) doppel-api
     Path: apps/dev/doppel-api/AGENTS.md
     doppel-api सेवा के K8s मैनिफ़ेस्ट प्रबंधन विशेषज्ञ।

  3) monitoring
     Path: infra/monitoring/AGENTS.md
     Prometheus + Grafana मॉनिटरिंग स्टैक का SRE विशेषज्ञ।

  ...

Select agent (number): 2

=== AI Tool ===
  1) claude
  2) codex
  3) print

Select tool: 1

→ claude सत्र doppel-api डायरेक्टरी में शुरू हुआ
→ एजेंट ने स्वचालित रूप से AGENTS.md और .ai-agents/context/ लोड किया
→ तुरंत काम शुरू!
```

**समानांतर निष्पादन (tmux):**

```bash
$ ./ai-agency.sh --multi

Select agents: 1,2,3   # PM + API + मॉनिटरिंग एक साथ चलाएं

→ tmux के 3 सत्र खुलेंगे
→ प्रत्येक विंडो में अलग-अलग एजेंट स्वतंत्र रूप से काम करेंगे
→ Ctrl+B N से विंडो बदलें
```

## उत्पाद सूची

इस प्रणाली में बनी फ़ाइलें और प्रत्येक का उद्देश्य:

| फ़ाइल | लक्ष्य | उद्देश्य |
|---|---|---|
| `HOW_TO_AGENTS.md` | AI | एजेंट द्वारा पढ़ी और निष्पादित की जाने वाली मेटा निर्देश पुस्तिका |
| `HOW_TO_AGENTS_PLAN.md` | मनुष्य/AI | डिज़ाइन योजना (यह संरचना क्यों है, इसकी पृष्ठभूमि) |
| `README.md` | मनुष्य | यह दस्तावेज़। मनुष्यों की समझ के लिए गाइड |
| `ai-agency.sh` | मनुष्य | एजेंट चयन → AI सत्र शुरू करने वाला लॉन्चर |
| `AGENTS.md` (प्रत्येक डायरेक्टरी) | AI | डायरेक्टरी-विशिष्ट एजेंट पहचान + नियम |
| `.ai-agents/context/*.md/json` | AI | पूर्व-व्यवस्थित डोमेन ज्ञान |
| `.ai-agents/skills/*/SKILL.md` | AI | मानकीकृत कार्य कार्यप्रवाह |
| `.ai-agents/roles/*.md` | AI/मनुष्य | भूमिका-विशिष्ट कॉन्टेक्स्ट लोडिंग रणनीति |

## मुख्य उपमा

```
              पारंपरिक विकास टीम          AI एजेंट टीम
              ──────────────────         ────────────────
 लीडर        PM (मनुष्य)                रूट AGENTS.md (PM एजेंट)
 सदस्य       डेवलपर N लोग              प्रत्येक डायरेक्टरी का AGENTS.md
 ऑनबोर्डिंग   Confluence/Notion         .ai-agents/context/
 कार्य मैनुअल  टीम विकी                 .ai-agents/skills/
 भूमिका       पद/R&R दस्तावेज़           .ai-agents/roles/
 टीम नियम    टीम कन्वेंशन दस्तावेज़      Global Conventions (इनहेरिटेंस)
 कार्यालय आना कार्यालय पहुंचना           सत्र शुरू → AGENTS.md लोड
 घर जाना     घर जाना (स्मृति बरकरार)    सत्र समाप्त (स्मृति नष्ट!)
 अगले दिन आना स्मृति बरकरार             .ai-agents/context/ लोड (स्मृति पुनर्स्थापित)
```

**मुख्य अंतर:** मनुष्य घर जाने के बाद भी याद रखता है, लेकिन AI हर बार भूल जाता है।
इसीलिए `.ai-agents/context/` मौजूद है — यह AI की **दीर्घकालिक स्मृति** की भूमिका निभाता है।
